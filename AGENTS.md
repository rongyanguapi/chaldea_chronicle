# AGENTS.md

## 项目通用规则

- 本项目是独立的纯 Flutter 工程，与 CamScanner 无业务、代码或平台耦合。参考外部项目规则或迁移 skill 时，必须移除 CamScanner 业务规则、内部平台、内部账号、内部接口、埋点平台等专属内容。
- 项目内新增或维护的 Codex skill 默认使用简体中文，包括 `SKILL.md` 正文、描述、提示语和面向开发者的说明。
- 除非用户明确要求写测试，否则不要新增或维护 `test/` 下的测试代码，也不要主动补单元测试、widget 测试、集成测试或 golden 测试。
- 新增代码注释、示例注释和文档注释默认使用简体中文，描述要简洁清晰。技术标识符、API 名称、命令、文件路径、Dart/Flutter 类型名和第三方库名称保持原文。复杂业务逻辑、协议解析、生命周期约束和非显然边界行为需要补充必要注释，避免大段业务代码完全无注释。
- `lib/pages/` 是业务代码主目录，具体业务默认以 page 为维度组织；不要使用 `lib/features/` 作为业务聚合层。
- 页面颜色优先使用 `lib/resource/app_colors.dart` 中已定义的 token。若当前没有合适 token，先在 `AppColors` 中新增语义清晰的颜色常量，再在页面中引用，不要在页面内直接写散落的 `Color(...)` 或 `Colors.*`。
- 如果用户没有明确要求编译验证，不主动运行 Flutter/Gradle/Android/iOS 构建任务；优先通过静态检查、差异检查、搜索引用和必要的轻量级分析完成验证。
- 根目录 `.workspace/` 是本地工作空间，只存放不需要随项目提交的临时资料、分析输出、草稿和中间产物；该目录必须被 Git 忽略，不要把其中内容作为项目交付物提交。
- 根目录 `doc/` 用于沉淀页面业务逻辑说明。页面文档按路由结构建目录，根路由 `/` 对应 `doc/routes/root/`，后续命名路由按路径分段建立子目录；每个页面目录内使用 `README.md` 描述该页面的职责、入口、核心状态、主要交互和边界行为。
